🚨
VOSK НЕ ЗАПУСКАЕТСЯ НА ESP32!

ESP32 имеет 520KB RAM и 240MHz. Vosk (даже small) требует ~200MB RAM и ARM Cortex уровня смартфона. ESP32 — только микрофон + Bluetooth. Vosk запускается на Android-телефоне.

🔌 Шаг 1 — Подключение INMP441 к ESP32

ℹ️
INMP441 vs ICS-43434

INMP441 — это тоже I2S MEMS микрофон. Выдаёт 24-бит данные в 32-бит словах, выровненных влево (MSB). Питание: только 3.3V (не 5V — убьёшь микрофон). Всенаправленный, SNR 61 dB, частотный диапазон 60–15000 Гц.

⚠️
Частые ошибки при подключении

1) Не перепутай WS и SCK — одинаково выглядят, но разные сигналы. 2) L/R обязательно на GND, иначе тишина. 3) Только 3.3V питание. 4) Если используешь макетную плату — длинные провода к SCK создают помехи, делай их короткими (<5см).

🎙️ Шаг 3 — I2S драйвер для INMP441

🧠
Что такое I2S и почему он нужен

I2S (Inter-IC Sound) — это цифровой протокол для передачи аудио между чипами. INMP441 выдаёт звук не как аналог через АЦП, а как цифровой поток: 3 провода — тактирование (SCK), сигнал выбора канала (WS/LRCK) и данные (SD). ESP32 имеет встроенный аппаратный I2S контроллер с DMA — это значит аудио пишется в память автоматически, без участия CPU.

src/i2s_mic.h — драйвер микрофона C++ 
#pragma once
#include "config.h"
#include "driver/i2s.h"

/*
 * I2S MEMS Microphone Driver для INMP441
 *
 * INMP441 особенности:
 *  - Выдаёт 24-битные данные в 32-битных словах
 *  - Данные ВЫРОВНЕНЫ ВЛЕВО (MSB first): [D23 D22 ... D0 | 0 0 0 0 0 0 0 0]
 *  - Для получения int16: сдвигаем вправо на 16 бит
 *  - Если L/R = GND → LEFT channel. L/R = 3.3V → RIGHT channel.
 */

namespace I2SMic {

// Инициализация I2S контроллера ESP32
bool begin() {
    i2s_config_t i2s_config = {
        .mode            = (i2s_mode_t)(I2S_MODE_MASTER | I2S_MODE_RX),
        .sample_rate     = SAMPLE_RATE,
        .bits_per_sample = I2S_BITS_PER_SAMPLE_32BIT,  // INMP441 → 32-бит слова
        .channel_format  = I2S_CHANNEL_FMT_ONLY_LEFT,  // L/R на GND → левый
        .communication_format = I2S_COMM_FORMAT_STAND_I2S,
        .intr_alloc_flags = ESP_INTR_FLAG_LEVEL1,
        .dma_buf_count   = DMA_BUF_CNT,
        .dma_buf_len     = DMA_BUF_LEN,
        .use_apll        = false,  // Audio PLL — для точности, но не нужно
        .tx_desc_auto_clear = false,
        .fixed_mclk      = 0
    };

    i2s_pin_config_t pin_config = {
        .bck_io_num   = I2S_SCK_PIN,
        .ws_io_num    = I2S_WS_PIN,
        .data_out_num = I2S_PIN_NO_CHANGE,  // TX не нужен
        .data_in_num  = I2S_SD_PIN
    };

    esp_err_t err = i2s_driver_install(I2S_NUM_0, &i2s_config, 0, NULL);
    if (err != ESP_OK) {
        Serial.printf("[I2S] driver_install failed: %d\n", err);
        return false;
    }

    err = i2s_set_pin(I2S_NUM_0, &pin_config);
    if (err != ESP_OK) {
        Serial.printf("[I2S] set_pin failed: %d\n", err);
        return false;
    }

    // Явно задаём тактирование (необходимо для стабильности)
    i2s_set_clk(I2S_NUM_0, SAMPLE_RATE, I2S_BITS_PER_SAMPLE_32BIT, I2S_CHANNEL_MONO);

    Serial.println("[I2S] Microphone INMP441 initialized OK");
    return true;
}

/*
 * Читает один DMA-буфер из I2S (DMA_BUF_LEN сэмплов = 512)
 *
 * rawBuf    — выходной буфер для RAW 32-bit данных
 * pcmBuf    — выходной буфер для 16-bit PCM (готово для Vosk/Bluetooth)
 * maxSamples — размер буферов
 *
 * Возвращает количество прочитанных сэмплов (обычно DMA_BUF_LEN)
 *
 * АЛГОРИТМ КОНВЕРТАЦИИ:
 *   INMP441 raw word (32 bit):
 *   [31..8] = 24-бит данные (MSB aligned)  [7..0] = 0x00
 *
 *   Шаг 1: интерпретируем как int32_t (знаковое)
 *   Шаг 2: >> 16 → получаем 16-bit со знаком (обрезаем младшие 16 нулей)
 *
 *   Почему >> 16, а не >> 8?
 *   INMP441: 24-bit данные НАЧИНАЮТСЯ с бита 31 (MSB).
 *   Биты 31-8 = данные. Биты 7-0 = нули.
 *   (int32_t)raw >> 16 сдвигает данные в позицию бит 15-0 → int16_t.
 */
size_t read(int32_t* rawBuf, int16_t* pcmBuf, size_t maxSamples) {
    size_t bytesRead = 0;
    
    // portMAX_DELAY = ждём бесконечно пока DMA не заполнится
    i2s_read(I2S_NUM_0,
             rawBuf,
             maxSamples * sizeof(int32_t),
             &bytesRead,
             portMAX_DELAY);
    
    size_t samplesRead = bytesRead / sizeof(int32_t);
    
    // Конвертация 32-bit → 16-bit PCM
    for (size_t i = 0; i < samplesRead; i++) {
        // Знаковый сдвиг вправо — сохраняет знаковый бит
        pcmBuf[i] = (int16_t)((int32_t)rawBuf[i] >> 16);
    }
    
    return samplesRead;
}

// Остановка I2S (для light sleep)
void suspend() { i2s_stop(I2S_NUM_0); }
void resume()  { i2s_start(I2S_NUM_0); }

} // namespace I2SMic

📡 Шаг 4 — Bluetooth транспорт

ℹ️
Почему BT Classic SPP, а не BLE?

BLE (Bluetooth Low Energy) имеет MTU ~512 байт и throughput ~100 кбит/с. Наш аудиопоток = 256 кбит/с — BLE не справится! BT Classic SPP (Serial Port Profile) даёт 300–700 кбит/с, что достаточно с запасом.

src/bt_transport.h — протокол передачи C++ 
#pragma once
#include "config.h"
#include "BluetoothSerial.h"

/*
 * Протокол пакетов VoiceMind Transport v1
 *
 * Формат каждого пакета:
 * ┌────────┬──────────┬────────┬──────────┬──────────────┬───────┐
 * │ 0xAA   │ SEQ[2]   │ TYPE   │ LEN[2]   │ PAYLOAD[LEN] │ CRC8  │
 * │ 1 байт │ 2 байта  │ 1 байт │ 2 байта  │ 0..1024 байт │ 1 байт│
 * └────────┴──────────┴────────┴──────────┴──────────────┴───────┘
 * Всего overhead: 7 байт на пакет
 *
 * CRC-8 (полином 0x31, Dallas/Maxim) считается по байтам 0..N-2
 */

namespace BTTransport {

BluetoothSerial SerialBT;
static uint16_t seqNum = 0;

// ─── CRC-8 (Dallas / Maxim) ──────────────────────────────────────
// Полином: P(x) = x^8 + x^5 + x^4 + 1 = 0x31
// Применяется к ВСЕМ байтам пакета ДО CRC (header + payload)
static uint8_t crc8(const uint8_t* data, size_t len) {
    uint8_t crc = 0x00;
    for (size_t i = 0; i < len; i++) {
        crc ^= data[i];
        for (int bit = 0; bit < 8; bit++) {
            if (crc & 0x80) crc = (crc << 1) ^ 0x31;
            else             crc = (crc << 1);
        }
    }
    return crc;
}

// Буфер для одного пакета: 1(start) + 2(seq) + 1(type) + 2(len) + 1024(payload) + 1(crc)
static uint8_t _pktBuf[7 + PKT_MAX_PAYLOAD + 1];

// ─── Инициализация BT ────────────────────────────────────────────
bool begin() {
    if (!SerialBT.begin(BT_DEVICE_NAME)) {
        Serial.println("[BT] Failed to start Bluetooth!");
        return false;
    }
    Serial.printf("[BT] Device '%s' is discoverable\n", BT_DEVICE_NAME);
    return true;
}

bool isConnected() { return SerialBT.hasClient(); }

// ─── Отправка пакета ─────────────────────────────────────────────
static void sendPacket(uint8_t type, const uint8_t* payload, uint16_t payloadLen) {
    if (!SerialBT.hasClient()) return;
    
    // Собираем заголовок
    _pktBuf[0] = PKT_START_BYTE;
    _pktBuf[1] = (seqNum >> 8) & 0xFF;   // SEQ high byte (big-endian)
    _pktBuf[2] = seqNum & 0xFF;           // SEQ low byte
    _pktBuf[3] = type;
    _pktBuf[4] = (payloadLen >> 8) & 0xFF; // LEN high byte
    _pktBuf[5] = payloadLen & 0xFF;        // LEN low byte
    
    // Копируем payload
    memcpy(&_pktBuf[6], payload, payloadLen);
    
    // Считаем CRC по всему (заголовок + payload)
    uint16_t totalBeforeCRC = 6 + payloadLen;
    _pktBuf[totalBeforeCRC] = crc8(_pktBuf, totalBeforeCRC);
    
    // Отправляем всё одним блоком
    SerialBT.write(_pktBuf, totalBeforeCRC + 1);
    seqNum++;
}

// ─── Аудио данные ────────────────────────────────────────────────
void sendAudio(const int16_t* pcm, size_t samples) {
    sendPacket(PKT_AUDIO,
               (const uint8_t*)pcm,
               (uint16_t)(samples * sizeof(int16_t)));
}

// ─── Статус (батарея, режим) ─────────────────────────────────────
void sendStatus(uint8_t battPct, bool isRec) {
    uint8_t payload[3] = { battPct, (uint8_t)isRec, 0x01 }; // v1
    sendPacket(PKT_STATUS, payload, sizeof(payload));
}

// ─── Heartbeat (keepalive каждую секунду) ────────────────────────
void sendHeartbeat() {
    uint8_t dummy = 0;
    sendPacket(PKT_HEARTBEAT, &dummy, 1);
}

} // namespace BTTransport

🚀 Шаг 5 — Главный файл main.cpp

Это сердце прошивки. Здесь всё связывается вместе: I2S читает аудио, BT отправляет, кнопка управляет, LED сигнализирует.

src/main.cpp — полная прошивка VoiceMind C++ 
#include "Arduino.h"
#include "config.h"
#include "i2s_mic.h"
#include "bt_transport.h"
#include "esp_sleep.h"

// ─── ГЛОБАЛЬНОЕ СОСТОЯНИЕ ─────────────────────────────────────────
volatile bool gIsRecording   = false;  // volatile: изменяется в прерывании
volatile bool gBtnPressed    = false;
uint32_t      gLastHeartbeat = 0;
uint32_t      gLastBtActivity= 0;

// Аудио буферы (статические — не в стеке, нет риска переполнения)
static int32_t rawBuf[DMA_BUF_LEN];   // 32-bit сырые данные с I2S (2048 байт)
static int16_t pcmBuf[DMA_BUF_LEN];   // 16-bit PCM для передачи  (1024 байт)

// ─── ПРЕРЫВАНИЕ КНОПКИ ───────────────────────────────────────────
// IRAM_ATTR — функция хранится в IRAM, выполняется быстро даже при cache miss
void IRAM_ATTR onButtonISR() {
    gBtnPressed = true;  // Только флаг! Обработка в loop()
}

// ─── ЧТЕНИЕ БАТАРЕИ ──────────────────────────────────────────────
/*
 * GPIO34 — ADC1 CH6 (12-bit, 0-4095 = 0..3.3V)
 * Делитель напряжения: R1=100kΩ, R2=100kΩ
 *   V_adc = V_bat × R2/(R1+R2) = V_bat / 2
 *   V_bat = V_adc × 2
 * Li-Po: полная = 4.2V, пустая = 3.0V
 * Берём 8 замеров и усредняем (убираем шум АЦП)
 */
uint8_t readBatteryPercent() {
    uint32_t sum = 0;
    for (int i = 0; i < 8; i++) sum += analogRead(BATT_ADC_PIN);
    float v_adc = (sum / 8.0f) * (3.3f / 4095.0f);
    float v_bat = v_adc * 2.0f;
    float pct   = (v_bat - 3.0f) / (4.2f - 3.0f) * 100.0f;
    return (uint8_t)constrain(pct, 0, 100);
}

// ─── ИНДИКАЦИЯ LED ───────────────────────────────────────────────
// Паттерны мигания: они не блокируют loop (используем millis)
void updateLED() {
    uint32_t now = millis();
    if (gIsRecording) {
        // Запись: быстрое мигание 4 Гц
        digitalWrite(LED_PIN, (now / 125) % 2);
    } else if (BTTransport::isConnected()) {
        // Подключён, не пишем: медленное мигание 0.5 Гц
        digitalWrite(LED_PIN, (now / 1000) % 2);
    } else {
        // Нет подключения: очень редко пикаем
        digitalWrite(LED_PIN, (now % 3000) < 100 ? HIGH : LOW);
    }
}

// ─── WATCHDOG: автостоп если BT молчит 30с ───────────────────────
void checkWatchdog() {
    if (gIsRecording && !BTTransport::isConnected()) {
        if (millis() - gLastBtActivity > 30000) {
            Serial.println("[WDG] BT timeout — stopping recording");
            gIsRecording = false;
        }
    }
}

// ─── SETUP ───────────────────────────────────────────────────────
void setup() {
    Serial.begin(115200);
    Serial.println("\n\n=== VoiceMind Firmware v1.0 ===");
    
    // GPIO
    pinMode(LED_PIN, OUTPUT);
    pinMode(BTN_PIN, INPUT_PULLUP);
    
    // Прерывание на кнопку: FALLING = нажатие (HIGH→LOW)
    attachInterrupt(digitalPinToInterrupt(BTN_PIN), onButtonISR, FALLING);
    
    // Инициализация I2S микрофона
    if (!I2SMic::begin()) {
        Serial.println("[ERR] I2S init failed! Check wiring.");
        // Мигаем быстро — ошибка
        while(true) { digitalWrite(LED_PIN, !digitalRead(LED_PIN)); delay(100); }
    }
    
    // Инициализация Bluetooth
    if (!BTTransport::begin()) {
        Serial.println("[ERR] BT init failed!");
        while(true) { digitalWrite(LED_PIN, !digitalRead(LED_PIN)); delay(200); }
    }
    
    Serial.println("[OK] VoiceMind ready!");
    Serial.printf("[OK] BT name: %s\n", BT_DEVICE_NAME);
    Serial.println("[OK] Press BTN to start recording");
    
    gLastBtActivity = millis();
}

// ─── LOOP ────────────────────────────────────────────────────────
void loop() {
    
    // 1. Обработка нажатия кнопки (из ISR)
    if (gBtnPressed) {
        gBtnPressed = false;
        delay(50);  // debounce 50мс
        gIsRecording = !gIsRecording;
        Serial.printf("[BTN] Recording: %s\n", gIsRecording ? "ON" : "OFF");
        
        if (BTTransport::isConnected()) {
            BTTransport::sendStatus(readBatteryPercent(), gIsRecording);
        }
    }
    
    // 2. Heartbeat (каждую секунду когда подключён)
    if (BTTransport::isConnected() && (millis() - gLastHeartbeat > HEARTBEAT_MS)) {
        BTTransport::sendHeartbeat();
        gLastHeartbeat = millis();
        gLastBtActivity = millis();
    }
    
    // 3. Watchdog
    checkWatchdog();
    
    // 4. LED индикация
    updateLED();
    
    // 5. Аудио — главный путь данных
    if (gIsRecording && BTTransport::isConnected()) {
        
        // Читаем DMA буфер (БЛОКИРУЮЩИЙ вызов: ждём ~32мс до заполнения)
        size_t samples = I2SMic::read(rawBuf, pcmBuf, DMA_BUF_LEN);
        
        if (samples > 0) {
            // Отправляем по Bluetooth
            BTTransport::sendAudio(pcmBuf, samples);
            gLastBtActivity = millis();
            
            // Отладка: выводим энергию каждые 100 буферов
            static uint32_t debugCount = 0;
            if (++debugCount % 100 == 0) {
                int64_t sumSq = 0;
                for (size_t i = 0; i < samples; i++) sumSq += (int64_t)pcmBuf[i] * pcmBuf[i];
                float rms = sqrt((double)sumSq / samples);
                Serial.printf("[AUDIO] RMS=%.0f samples=%d\n", rms, samples);
            }
        }
        
    } else if (!gIsRecording) {
        // Не записываем — небольшая задержка чтобы не жечь CPU
        delay(10);
    } else {
        // Записываем но нет BT — ждём подключения
        delay(100);
    }
}

∑ Шаг 8 — Математика которую ты реально используешь

1. PCM — что это за числа
Частота дискретизации: fs = 16 000 Гц → 16 000 замеров/с
Каждый сэмпл: знаковое int16 ∈ [-32768, +32767]
Поток данных: R = 16000 × 2 байта = 32 000 байт/с = 256 кбит/с
2. Конвертация INMP441: 32bit → 16bit
INMP441 выдаёт: [бит 31..8] = данные (24 бита) | [бит 7..0] = 0x00
Конвертация: sample16 = (int32_t)raw >> 16
Почему 16, а не 8? Сдвиг 16 = делим данные пополам, получаем 16-bit диапазон
3. RMS Energy (для VAD)
E_rms = √( (1/N) × Σᵢ xᵢ² )
Где N = 512 (размер фрейма), xᵢ — сэмпл
Типичные значения: тишина ≈ 100-400, речь ≈ 800-8000
Порог VAD: thresh = 500
4. EMA (сглаживание для VAD)
E_smooth[t] = α × E_rms[t] + (1 - α) × E_smooth[t-1]
α = 0.3 → временная константа TC ≈ 89 мс
Меньше α → медленнее реагирует, больше сглаживания
5. CRC-8 (целостность пакетов)
Полином: P(x) = x⁸ + x⁵ + x⁴ + 1 → коэффициент 0x31
Для каждого байта: CRC = CRC XOR byte, затем 8 итераций:
  если старший бит = 1: CRC = (CRC << 1) XOR 0x31
  иначе: CRC = CRC << 1
Итог: 1 байт защиты на до 2 KB данных

🤖 Шаг 9 — Vosk: где и как он работает

✅
Vosk запускается на Android, получает PCM поток от ESP32

ESP32 шлёт сырые PCM данные (16kHz, 16-bit, mono) через BT. Android получает, собирает в чанки по 512-4096 сэмплов, подаёт в Vosk. Vosk возвращает JSON с текстом и временными метками слов.

ШагЧто происходит
MFCCMel-Frequency Cepstral Coefficients — преобразует PCM в 40 коэффициентов, имитирующих восприятие слуха человека (логарифмическая шкала частот)
DNNНейросеть определяет вероятность каждой фонемы для текущего фрейма (~25мс окно)
Viterbi HMMАлгоритм динамического программирования находит наиболее вероятную последовательность слов через языковую модель (n-gram)
